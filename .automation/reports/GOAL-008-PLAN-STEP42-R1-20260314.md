# GOAL-008 Step 4.2 R1 Planning Report (2026-03-14)

## EN
### Summary
Planned a focused enhancement pass on top of Step 4.2 so the trading-plan workflow is usable in day-to-day operation. The scope is limited to editing existing plans, hard deletion, deterministic defaulting for stop-loss/take-profit/target prices from saved agent history, and a cross-stock trading-plan board that is visible even before selecting a symbol.

### Why This R1 Is Needed
- The first Step 4.2 loop only supported draft -> save, so users could not correct plans after saving.
- Plans exposed `TriggerPrice` / `InvalidPrice` / `StopLossPrice`, but not explicit `TakeProfitPrice` or `TargetPrice`.
- Current plans were only visible inside the active stock workspace.
- The UX gap forces users to re-open individual stocks to inspect plan state.

### Planned Backend Changes
1. Extend `TradingPlan` and plan DTOs with `TakeProfitPrice` and `TargetPrice`.
2. Expand `TradingPlanDraftService` to derive price defaults deterministically from saved agent history with precedence:
   - commander `chart.takeProfitPrice` / `chart.targetPrice`
   - financial `metrics.institutionTargetPrice`
   - trend `forecast[].price` extreme fallback
3. Extend `TradingPlanService` to support optional global listing and hard delete.
4. Extend `StocksModule.cs` with:
   - `GET /api/stocks/plans?take=...` for board mode
   - `DELETE /api/stocks/plans/{id}`
5. Keep local DB compatibility additive for legacy `TradingPlans` tables.

### Planned Frontend Changes
1. Add an always-visible `交易计划总览` card in `StockInfoTab.vue`.
2. Reuse the existing plan modal for both create and edit flows.
3. Add per-plan `编辑` and `删除` actions in the current-stock list.
4. Display `止损` / `止盈` / `目标` pills in both the board and the current-stock list.
5. Refresh both the per-stock list and the global board after create, edit, or delete.

### Validation Plan
- Backend targeted tests for draft price derivation, create persistence, update guards, and delete behavior.
- Frontend `StockInfoTab.spec.js` updates for board loading, extra price fields, edit flow, and delete flow.
- Keep validation unit-test-first.

## ZH
### 摘要
本次规划是在 Step 4.2 之上做一个可用性增强补丁，目标只聚焦四件事：已保存交易计划可编辑、可硬删除、止损/止盈/目标价能基于已保存 Agent 历史做确定性默认填充，以及在不选股时也能看到跨股票的交易计划总览。

### 为什么要做这个 R1
- 第一版 Step 4.2 只有“起草 -> 保存”，保存后无法继续修正计划。
- 计划字段里只有 `TriggerPrice` / `InvalidPrice` / `StopLossPrice`，没有明确的 `TakeProfitPrice` / `TargetPrice`。
- 现有计划只能在当前股票 workspace 里看见。
- 用户如果想看全局计划状态，必须重新点进每只股票，操作成本高。

### 后端计划改动
1. 给 `TradingPlan` 和相关 DTO 增加 `TakeProfitPrice`、`TargetPrice`。
2. 扩展 `TradingPlanDraftService`，按确定性优先级从已保存历史中推导默认价格：
   - commander 的 `chart.takeProfitPrice` / `chart.targetPrice`
   - financial 的 `metrics.institutionTargetPrice`
   - trend 的 `forecast[].price` 极值兜底
3. 扩展 `TradingPlanService`，支持全局列表查询和硬删除。
4. 在 `StocksModule.cs` 增加：
   - `GET /api/stocks/plans?take=...` 作为总览模式
   - `DELETE /api/stocks/plans/{id}`
5. 继续保持对旧版 `TradingPlans` 本地库的增量兼容。

### 前端计划改动
1. 在 `StockInfoTab.vue` 增加始终可见的 `交易计划总览` 卡片。
2. 复用现有计划弹窗，同时支持新建和编辑。
3. 在当前股票计划列表中加入 `编辑`、`删除` 操作。
4. 在总览和当前股票列表都显示 `止损` / `止盈` / `目标` 标签。
5. 在创建、编辑、删除后同时刷新当前股票列表和全局总览。

### 验证计划
- 后端定向单测覆盖草稿价格推导、创建持久化、更新约束和删除行为。
- 前端更新 `StockInfoTab.spec.js`，覆盖总览加载、新增价格字段、编辑流程、删除流程。
- 按“先单测，后续再做 UI 验证”的顺序执行。
