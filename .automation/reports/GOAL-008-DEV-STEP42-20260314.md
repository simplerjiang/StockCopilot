# GOAL-008 Step 4.2 Development Report (2026-03-14)

## EN
### Scope
- Completed the minimal human-in-the-loop trading-plan loop on top of the existing stock multi-agent workflow.
- Kept the authoritative draft source on the backend by reusing saved `StockAgentAnalysisHistory` commander output instead of adding a second LLM extraction pass.

### Development
- Backend domain and persistence:
  - Added `TradingPlan`, `TradingPlanDirection`, and `TradingPlanStatus`.
  - Updated `AppDbContext` with `DbSet<TradingPlan>`, enum string conversion, indexes, and the `StockAgentAnalysisHistory` relationship.
  - Registered and exposed trading-plan services and endpoints in `StocksModule.cs`:
    - `GET /api/stocks/plans?symbol=...`
    - `GET /api/stocks/plans/{id}`
    - `POST /api/stocks/plans/draft`
    - `POST /api/stocks/plans`
    - `PUT /api/stocks/plans/{id}`
    - `POST /api/stocks/plans/{id}/cancel`
- Draft generation:
  - Added `ITradingPlanDraftService` / `TradingPlanDraftService`.
  - Drafts are built from saved commander history only.
  - `TriggerPrice`, `InvalidPrice`, and `StopLossPrice` are filled only from deterministic chart fields such as `chart.breakoutPrice`, `chart.supportPrice`, and `chart.stopLossPrice`.
  - Natural-language-only trigger text is not converted into numeric prices.
- Save flow and watchlist linkage:
  - Added `ITradingPlanService` / `TradingPlanService` for list/get/create/update/cancel.
  - Creating a plan saves it as `Pending` and immediately calls `IActiveWatchlistService.UpsertAsync(symbol, name, "trading-plan", $"plan:{id}", true)`.
- Local database compatibility:
  - Added `TradingPlanSchemaInitializer` and generated migration `AddTradingPlansForStep42`.
  - Because the local database already contained a legacy `TradingPlans` table from older work, the initializer and migration were extended to add missing Step 4.2 columns/indexes instead of assuming a clean create.
  - Added compatibility defaults for legacy non-null columns `PlanKey` and `Title` so new Step 4.2 inserts do not fail on old schemas.
- Frontend flow:
  - `StockAgentPanels.vue` now shows `基于此分析起草交易计划` only when commander data is available and successful, and emits `draft-plan` to the parent.
  - `StockInfoTab.vue` now keeps per-symbol plan draft/loading/error/list state, opens a backend-driven draft modal, lets the user edit prices/text, saves the `Pending` plan, and refreshes the local current-plan section.
  - Added a `当前交易计划` section that shows the latest plan cards for the active stock.
- Tests:
  - Added backend `TradingPlanServicesTests` for deterministic draft mapping, nullable price behavior, watchlist auto-upsert, and non-pending update rejection.
  - Updated `StockAgentPanels.spec.js` and `StockInfoTab.spec.js` to cover the commander CTA and the draft/save refresh flow.

### 开发
- 后端领域与持久化：
  - 新增 `TradingPlan`、`TradingPlanDirection`、`TradingPlanStatus`。
  - 更新 `AppDbContext`，加入 `DbSet<TradingPlan>`、枚举字符串映射、索引以及与 `StockAgentAnalysisHistory` 的关联。
  - 在 `StocksModule.cs` 注册并开放交易计划接口：
    - `GET /api/stocks/plans?symbol=...`
    - `GET /api/stocks/plans/{id}`
    - `POST /api/stocks/plans/draft`
    - `POST /api/stocks/plans`
    - `PUT /api/stocks/plans/{id}`
    - `POST /api/stocks/plans/{id}/cancel`
- 草稿生成：
  - 新增 `ITradingPlanDraftService` / `TradingPlanDraftService`。
  - 草稿只从已保存的 commander 历史结果构建。
  - `TriggerPrice`、`InvalidPrice`、`StopLossPrice` 只从 `chart.breakoutPrice`、`chart.supportPrice`、`chart.stopLossPrice` 这类确定性图表字段读取。
  - 纯自然语言里的价格描述不会被臆测成数值落库。
- 保存与白名单联动：
  - 新增 `ITradingPlanService` / `TradingPlanService`，支持查询、创建、更新、取消。
  - 创建计划时固定保存为 `Pending`，并立即调用 `IActiveWatchlistService.UpsertAsync(symbol, name, "trading-plan", $"plan:{id}", true)` 自动进入 `ActiveWatchlist`。
- 本地数据库兼容：
  - 新增 `TradingPlanSchemaInitializer`，并生成 `AddTradingPlansForStep42` migration。
  - 由于本地数据库里已经存在旧版 `TradingPlans` 表，本轮没有假设全新建表，而是补齐 Step 4.2 缺失列与索引。
  - 额外为旧表的非空列 `PlanKey`、`Title` 增加默认约束，避免新 Step 4.2 插入被旧 schema 拦截。
- 前端流程：
  - `StockAgentPanels.vue` 仅在 commander 成功结果存在时显示“基于此分析起草交易计划”按钮，并向父组件发出 `draft-plan` 事件。
  - `StockInfoTab.vue` 为每个 symbol 维护独立的计划草稿/保存/错误/列表状态，打开后端生成的草稿弹窗，允许用户补录价格与文本，保存后刷新当前计划区块。
  - 页面新增 `当前交易计划` 区块，展示当前股票最近计划卡片。
- 测试：
  - 新增后端 `TradingPlanServicesTests`，覆盖确定性草稿映射、价格缺失保空、自动进白名单、非 `Pending` 禁止编辑。
  - 更新 `StockAgentPanels.spec.js` 与 `StockInfoTab.spec.js`，覆盖 commander 起草按钮和 draft/save 刷新链路。
