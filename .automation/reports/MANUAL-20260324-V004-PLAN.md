# MANUAL-20260324 v0.0.4 PLAN

## EN

### Scope

- Review the published `v0.0.3` release notes and make them suitable for external users.
- Define the first concrete task breakdown for the next version after `v0.0.3`.
- Keep the next version focused on product acceptance and the few unfinished slices that directly improve the Stock Copilot product.

### Release Notes Decision

- `v0.0.3` should present itself as the first usable Stock Copilot desktop build, not just as an internal changelog dump.
- The release note body should therefore highlight:
  - completed Stock Copilot session workflow,
  - trading-plan draft workflow fix,
  - packaged desktop availability,
  - installer and portable artifacts,
  - compare link for deeper technical review.

### Proposed v0.0.4 Breakdown

1. `GOAL-AGENT-002-R4` as the primary release slice.
   - Add measurable Copilot acceptance metrics.
   - Build replayable session baselines.
   - Expose product-level quality signals for tool efficiency, evidence coverage, local-first hit rate, traceability, and action quality.

2. `GOAL-015` test closure as the supporting quality slice.
   - Finish the remaining Browser MCP and UI acceptance for the fundamental snapshot / detail-loading flow.
   - Make sure the richer stock detail facts are not only implemented but also accepted as a stable user-facing surface.

3. `GOAL-017-R1` as the next backend foundation slice after R4 is stable.
   - Start the normalized quant input and signal contract work.
   - Keep this behind the product acceptance slice so the team does not deepen quant complexity before Copilot quality becomes measurable.

### Recommended Release Framing For v0.0.4

- Theme: `Copilot quality and acceptance visibility`.
- Main promise: the product should not only work, but also prove that it works better.
- Non-goal: do not expand raw model scope faster than the runtime can measure, replay, and audit it.

  - Result: to be run after edits.

## ZH

### 范围

- 复核已发布的 `v0.0.3` Release 文案，并把它补成适合对外展示的版本。
- 为 `v0.0.3` 之后的下一版本给出第一版可执行任务拆分。
- 下一版本继续聚焦真正影响 Stock Copilot 产品体验的未完成主线，不再分散到过多并行方向。

### Release 文案决策

- `v0.0.3` 应该被描述为“第一个可用的 Stock Copilot 桌面版”，而不是只挂一个自动生成的 changelog。
- 因此 Release 正文应突出：
  - 已完成的 Copilot 会话化工作流，
  - 已修复的交易计划草稿弹窗链路，
  - 已可下载使用的桌面打包版，
  - 安装包与便携包产物，
  - 便于技术回看的 compare 链接。

### 建议的 v0.0.4 拆分

1. 把 `GOAL-AGENT-002-R4` 作为主发布切片。
   - 增加可度量的 Copilot 产品验收指标。
   - 建立可 replay 的 session baseline。
   - 暴露工具效率、evidence 覆盖率、local-first 命中率、traceability、action 质量等产品级质量信号。

2. 把 `GOAL-015` 的测试收口作为配套质量切片。
   - 补完基本面快照 / 详情页加载链路剩余的 Browser MCP 与 UI 验收。
   - 确保这块 richer stock detail facts 不只是“代码已实现”，而是已经成为稳定的用户界面能力。

3. 把 `GOAL-017-R1` 作为 R4 稳定后的下一个底座切片。
   - 启动量化归一化输入层与 signal contract 工作。
   - 继续保持优先级在产品验收之后，避免在 Copilot 质量尚不可度量时先扩大量化复杂度。

### 建议的 v0.0.4 对外定位

- 主题：`Copilot 质量与验收可见性`。
- 核心承诺：产品不只是能跑起来，还要能证明这次改动让它更好。
- 非目标：不在 runtime 尚未完成可度量、可 replay、可审计前，继续盲目扩张原始模型能力面。
