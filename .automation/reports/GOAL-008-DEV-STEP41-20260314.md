# GOAL-008 Step 4.1 Development Report (2026-03-14)

## EN
### Scope
- Closed the remaining GOAL-008 Step 4.1 reviewer blockers for high-frequency quote monitoring.
- Kept the functional change backend-focused, then revalidated the existing stock-terminal frontend and Edge flow against the backend-served build.

### Development
- Added `ActiveWatchlist` as the dedicated source-of-truth table for symbols that need high-frequency monitoring.
- Added `ActiveWatchlistService` to normalize symbols and manage upsert, touch, enable/disable, sync timestamp, and delete operations.
- Added `HighFrequencyQuoteService` background worker plus `HighFrequencyQuoteOptions`.
- Extended `ChinaAStockMarketClock` from weekday/time-window gating to a holiday-aware 2026 China A-share closure calendar based on the official SSE closure schedule, so legal holidays no longer trigger the high-frequency worker.
- Replaced the `EastmoneyStockCrawler.GetIntradayMessagesAsync(...)` empty stub with a real Eastmoney `details/get` implementation that maps latest tape details into `IntradayMessageDto` records with explicit `东方财富逐笔` source tagging.
- Hardened `HighFrequencyQuoteService` so intraday-message fetch failures degrade to an empty message set without dropping the quote/minute persistence path for the same symbol.
- Added watchlist endpoints in `StocksModule`:
  - `GET /api/stocks/watchlist`
  - `POST /api/stocks/watchlist`
  - `POST /api/stocks/watchlist/{symbol}/touch`
  - `DELETE /api/stocks/watchlist/{symbol}`
- Updated EF model, schema initializer, and migration `AddActiveWatchlistForStep41` so existing local databases and fresh databases both receive the new table/indexes.
- Updated `frontend/scripts/edge-check-goal013.mjs` so browser validation no longer incorrectly waits for a second market-news response after symbol search; the script now accepts either initial root-level market-news load or already-rendered DOM state.
