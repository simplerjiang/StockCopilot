## EN
### Summary
Completed GOAL-009-R1 as the first local-first market sentiment and sector rotation slice. The backend now persists market sentiment snapshots, sector rotation snapshots, and leader ladders; exposes `/api/market/sentiment/latest`, `/api/market/sentiment/history`, `/api/market/sectors`, `/api/market/sectors/{sectorCode}`, and `/api/market/sectors/{sectorCode}/leaders`; and serves a new top-level `情绪轮动` tab in the frontend.

### Implementation
- Backend:
  - Added `MarketSentimentSnapshot`, `SectorRotationSnapshot`, and `SectorRotationLeaderSnapshot` entities and wired them into `AppDbContext`.
  - Added `MarketSentimentSchemaInitializer` and `SectorRotationWorker`.
  - Added `MarketModule`, `SectorRotationIngestionService`, `SectorRotationQueryService`, `EastmoneySectorRotationClient`, DTO models, and endpoint mapping under `/api/market/*`.
  - Added stable fallbacks for GOAL-009 ingestion:
    - non-trading days now fall back to the latest trading date before querying limit-up / limit-down / broken-board pools;
    - sector turnover concentration now uses total market turnover as denominator and clamps to `0-100%`;
    - transient Eastmoney upstream failures now degrade per source instead of crashing first-time sync with HTTP 500.
- Frontend:
  - Added `frontend/src/modules/market/MarketSentimentTab.vue`.
  - Added the top-level `情绪轮动` tab in `frontend/src/App.vue`.
  - Added frontend tests for tab rendering, initial loading, and filter switching.
- Tests:
  - Added `SectorRotationQueryServiceTests.cs`.
  - Added `SectorRotationIngestionServiceTests.cs` coverage for trading-date fallback, turnover concentration, and partial-upstream-failure degradation.

### Issues Found And Fixed During Validation
- Initial live `/api/market/*` validation reproduced `500` when snapshot tables were empty.
- Root cause: the first-sync path called Eastmoney directly and let a single upstream `HttpRequestException` (`response ended prematurely`) abort the entire request.
- Fix: degraded each upstream source independently inside `SectorRotationIngestionService`, so partial source failure now yields stable zero/empty-state payloads instead of a server error.

### Residual Risk
- When all Eastmoney sector sources are temporarily unavailable, the frontend will show a valid empty state and zero-value sentiment summary instead of crashing. That is intentional for stability, but R2 should improve cache reuse and stale snapshot preference so users see the latest good snapshot rather than a fully empty board.
- Current live weekend data still depends on source availability and may legitimately show limited pool metrics outside trading hours.

### Reviewer Follow-up Fixes

- Fixed the `style` board source mapping to match the validated Eastmoney R1 plan source, so `SectorBoardTypes.Style` no longer points at a mismatched filter.

### 验证期间发现并修复的问题
- 在清空快照表后，首次调用 live `/api/market/*` 稳定复现了 `500`。
- 根因：首次同步路径直接请求东财，任一上游 `HttpRequestException`（例如 `response ended prematurely`）都会中断整条请求链。
- 修复：把 `SectorRotationIngestionService` 改为按数据源粒度降级，单个源失败时返回 fallback 值，整接口继续返回稳定 payload，而不是抛 500。

### 剩余风险
- 当东财板块榜源在同一时段全部不可用时，前端当前会展示合法空态和零值情绪摘要，而不是崩溃。这是本轮为了稳定性做的有意降级；后续 R2 更适合补“优先返回最近一份成功快照”的缓存策略，避免用户看到完全空表。
- 周末或非交易时段的 live 情绪口径仍受上游可用性影响，出现部分池子为零属于可接受现象。
