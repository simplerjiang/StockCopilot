# GOAL-013 Step 2.6 Development Report (2026-03-13)

## EN
### Scope
- Cleaned polluted market-news inputs by switching to a finance-only Sina stream and dropping low-quality self-media sources before persistence.
- Replaced dead overseas RSS feeds with five verified active finance and macro sources.
- Added a searchable, paginated archive API and the new `全量资讯库` tab for browsing AI-processed local facts across market, sector, and stock levels.

### Development
- Backend source cleanup:
  - `LocalFactIngestionService` now uses Sina `pageid=153&lid=2509` instead of the mixed feed that was leaking social noise into market context.
  - Added `BlockedSourceKeywords` and `IsBlockedSource(...)` so entries whose `Source` contains `自媒体` are dropped from both domestic market items and RSS aggregation.
  - Replaced stale WSJ / NYT feed definitions with CNBC, Seeking Alpha, CoinTelegraph, TechCrunch, and The Hill.
- Backend archive query path:
  - Added `LocalNewsArchiveItemDto` and `LocalNewsArchivePageDto`.
  - Extended `IQueryLocalFactDatabaseTool` with `QueryArchiveAsync(...)` that merges `LocalStockNews` and `LocalSectorReports`, supports keyword / level / sentiment filters, and applies stable reverse-chronological ordering with pagination.
  - Added `GET /api/news/archive` in `StocksModule.cs`, returning `400` for invalid filters.
- Frontend archive workspace:
  - Added `NewsArchiveTab.vue` and registered the `全量资讯库` tab in `App.vue`.
  - Implemented keyword, level, and sentiment filters; pagination; translated-title-first rendering; original-title secondary text; badges for level, sentiment, target, and AI tags; plus source jump-out buttons.
- Tests and browser flow:
  - Added blocked-source coverage in `LocalFactIngestionServiceTests`.
  - Updated RSS parser tests to the new active feed set.
  - Added archive merge/filter/pagination coverage in `QueryLocalFactDatabaseToolTests`.
  - Added `NewsArchiveTab.spec.js` and extended the GOAL-013 Edge script to validate archive loading and search.
