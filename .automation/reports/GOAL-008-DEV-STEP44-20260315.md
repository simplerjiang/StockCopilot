# GOAL-008 Step 4.4 Development Report (2026-03-15)

## EN
### Summary
Completed Step 4.4 of the trading-plan engine. The backend now runs a separate semantic review worker over recent local stock news for enabled watchlist symbols, moves threatened `Pending` plans into `ReviewRequired`, records structured review metadata in `TradingPlanEvents`, and allows a human to move the plan back to `Pending` through a new resume action. The frontend now renders `待复核`, exposes the linked review reason/news context, and provides a `恢复观察` action.

### Implementation
- Backend:
  - Added `TradingPlanStatus.ReviewRequired`.
  - Added `TradingPlanEventType.NewsReviewed`, `ReviewRequired`, and `ReviewCleared`.
  - Added `TradingPlanReviewOptions`, `TradingPlanReviewWorker`, and `TradingPlanReviewService`.
  - Reused `ILlmService.ChatAsync(...)` with a strict JSON-only prompt returning `isPlanThreatened`, `reason`, and `confidence`.
  - Kept Step 4.4 isolated from Step 4.3; only the separate worker runs semantic review.
  - Added `POST /api/stocks/plans/{id}/resume` and backend `ResumeAsync(...)` to move `ReviewRequired -> Pending`.
  - Stored structured review metadata inside `TradingPlanEvents.MetadataJson` for auditability and per-plan/per-news deduplication.
- Legacy DB compatibility:
  - Mapped legacy `TradingPlanEvents` columns `VersionId`, `Strategy`, `Reason`, and `CreatedAt` in the EF entity.
  - Filled those fields for new Step 4.3/4.4/manual-review event writers.
  - Extended `TradingPlanSchemaInitializer` to add these columns when they are missing in another local database.
- Frontend:
  - Added `tradingPlanReview.js` helper module for review-status normalization and metadata parsing.
  - Updated `StockInfoTab.vue` to render status badges with `观察中 / 待复核 / 已触发 / 已失效 / 已取消` labels.
  - Added review-detail rendering in both the global board and the current-plan card.
  - Added `恢复观察` action for `ReviewRequired` plans.
- Tests:
  - Added backend `TradingPlanReviewServiceTests.cs`.
  - Extended `TradingPlanServicesTests.cs` with `ReviewRequired` edit/resume coverage and tolerant status parsing.
  - Added frontend `tradingPlanReview.test.js`.
