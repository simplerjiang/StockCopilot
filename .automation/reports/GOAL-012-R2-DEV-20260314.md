  - Result: pass

  - Result: pass
  - Command: Browser MCP on backend-served `http://localhost:5119/`
  - Result: stock query for `600519` loaded live data successfully; `分时图` and `月K图` both switched successfully; active chart pane rendered `canvas` nodes and the placeholder disappeared
- Regression follow-up:
  - Fixed K-line timestamps from Unix seconds to Unix milliseconds so the x-axis no longer falls into 1970/1971 on `klinecharts`.
  - Normalized minute-line `volume` from cumulative values to per-bar deltas, keeping display units in hands.
  - Revalidated live higher-timeframe payloads: `interval=month` returned 60 K-lines and `interval=year` returned 26 K-lines; Browser MCP confirmed `月K图` and `年K图` both render without the placeholder.
- Issues / risks:
  - `klinecharts` is still a beta dependency (`10.0.0-beta1`), so future UI/browser validation should continue to focus on crosshair behavior, pane layout, and runtime regressions after dependency bumps.
  - Browser session logs still contained two stale historical `GET /api/stocks/plans?take=20 => 500` entries from an earlier page state, but fresh network traffic for the current backend-served run returned `200` and backend logs showed no new exception records.
- Conclusion:
  - The controlled replacement is technically viable, is now the active chart path, and remains isolated behind the existing adapter boundary.
