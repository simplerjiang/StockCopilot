# OPS-008 LLM Channel Switching Development Report (2026-03-14)

## EN
### Scope
- Added active LLM channel switching so the system can toggle between the existing default OpenAI-compatible endpoint and the official Gemini OpenAI-compatible endpoint.
- Preserved the secret-splitting model: tracked defaults stay in `llm-settings.json`, while API keys remain only in ignored local storage.
- Updated the admin UI so the active endpoint can be switched without editing files manually.

### Development
- Backend settings store:
  - Added `activeProviderKey` to the LLM settings document.
  - Split provider key from provider type with `providerType`, so keys like `default` and `gemini_official` can both route through the same `OpenAiProvider` implementation.
  - Added active-provider APIs on `ILlmSettingsStore` / `JsonFileLlmSettingsStore` and migrated legacy `openai` settings to `default` automatically.
- Runtime routing:
  - `LlmService` now resolves the effective provider key first, then dispatches by `providerType`.
  - Internal business flows that should follow the global switch now default to `active` instead of hardcoded `openai`.
  - Updated app defaults and stock/governance fallback paths accordingly.
- Admin UI:
  - `AdminLlmSettings.vue` now exposes both provider editing (`default`, `gemini_official`) and a separate active-channel switcher.
  - Added frontend tests covering the active-provider switch endpoint.
- Local secrets:
  - Seeded ignored `App_Data/llm-settings.local.json` with local-only entries for `default` and `gemini_official`.
  - No tracked file was used to store API keys.

### Validation
- `dotnet test backend/SimplerJiangAiAgent.Api.Tests/SimplerJiangAiAgent.Api.Tests.csproj --filter "FullyQualifiedName~LlmServiceTests|FullyQualifiedName~JsonFileLlmSettingsStoreTests|FullyQualifiedName~OpenAiProviderTests"` -> passed (11/11)
- `npm --prefix frontend run test:unit -- src/modules/admin/AdminLlmSettings.spec.js` -> passed (4/4)

### Notes
- The new official Gemini channel is configured as an OpenAI-compatible provider with base URL `https://generativelanguage.googleapis.com/v1beta/openai/`.
- Secrets remain local-only; repository defaults contain non-sensitive routing metadata only.

## ZH
### 范围
- 增加了全局 LLM 激活通道切换能力，可在现有默认 OpenAI 兼容端点与官方 Gemini OpenAI 兼容端点之间切换。
- 保持原有“默认配置与密钥分离”的模式：受控默认参数留在 `llm-settings.json`，API Key 只留在被忽略的本地文件。
- 更新管理员页面，使切换激活通道不再需要手动改文件。

### 开发
- 后端配置存储：
  - 为 LLM 配置文档新增 `activeProviderKey`。
  - 新增 `providerType`，把“配置键”和“底层协议类型”拆开，使 `default`、`gemini_official` 都能复用同一个 `OpenAiProvider` 实现。
  - 在 `ILlmSettingsStore` / `JsonFileLlmSettingsStore` 中新增激活通道读取与切换能力，并自动把历史 `openai` 配置迁移视作 `default`。
- 运行时路由：
  - `LlmService` 现在会先解析生效的 provider key，再按 `providerType` 选用底层实现。
  - 需要跟随全局切换的内部业务流已不再硬编码 `openai`，而改为默认走 `active`。
  - 同步更新了 app 默认配置与股票/来源治理链路的回退 provider。
- 管理端 UI：
  - `AdminLlmSettings.vue` 现在同时支持编辑 provider 配置（`default`、`gemini_official`）和切换全局激活通道。
  - 新增前端单测覆盖激活通道切换接口。
- 本地密钥：
  - 已在被忽略的 `App_Data/llm-settings.local.json` 中补齐 `default` 与 `gemini_official` 的本地专用密钥条目。
  - 没有把 API Key 写入任何 tracked 文件。
