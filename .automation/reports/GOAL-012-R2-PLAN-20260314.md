# GOAL-012-R2 KLineCharts Controlled Replacement Plan (2026-03-14)

## EN
### Summary
Created the next Dev2-only workstream after GOAL-012-R1. The PM direction is now explicit: replace the current chart engine rather than continue generic investigation. Based on the finished research and current repo structure, the only viable replacement candidate is `klinecharts`, and the migration must stay inside the existing `frontend/src/modules/stocks/charting/**` adapter boundary so Dev1's Step 4.2+ trading-plan work remains untouched.

### Why This Is The Next Step
- `TradingView-vue` is not suitable for the current Vue 3 + Vite codebase.
- `vue-tradingview-widgets` is a widget embedding path, not the right engine for first-party `minuteLines` and `kLines` rendering.
- `vue3-apexcharts` is acceptable for generic dashboards but not the preferred stock-terminal core.
- `klinecharts` is the only replacement candidate with a realistic path for built-in indicators, line-drawing models, and future strategy overlays.
- The repo already has a safe switch seam in:
  - `frontend/src/modules/stocks/StockCharts.vue`
  - `frontend/src/modules/stocks/charting/chartViews.js`
  - `frontend/src/modules/stocks/charting/useStockChartAdapter.js`

### Scope Boundaries
Dev2 may change only:
- `frontend/src/modules/stocks/StockCharts.vue`
- `frontend/src/modules/stocks/charting/**`
- `frontend/package.json`
- necessary frontend test files

Dev2 must not change:
- backend `TradingPlan` entity/API flow
- Step 4.2 draft/save logic
- `StockAgentAnalysisHistory` contract
- Step 4.3-4.5 backend design

### Implementation Plan
1. Add `klinecharts` as an exact pinned dependency.
2. Keep the parent component contract unchanged:
   - `minuteLines`
   - `kLines`
   - `basePrice`
   - `interval`
   - `aiLevels`
   - `update:interval`
3. Introduce a dedicated `klinecharts` adapter under `charting/**` instead of moving engine logic back into `StockCharts.vue`.
4. Preserve the single-tab terminal UX:
   - `分时图`
   - `日K图`
   - `月K图`
   - `年K图`
5. First migrate existing capabilities only:
   - base price / minute rendering
   - K-line rendering
   - AI support / breakout overlays
   - current hover/crosshair information
6. Add explicit extension seams for:
   - overlay registry
   - indicator registry
   - marker / signal layer API
7. Produce a final conclusion report stating whether `klinecharts` is ready for production replacement or should remain an isolated spike with rollback.

### Risks
- Current npm metadata shows `klinecharts` latest as `10.0.0-beta1`, so version pinning and rollback discipline are mandatory.
- If minute-line behavior, interaction parity, or overlay extensibility are worse than the current implementation, Dev2 must report a rollback recommendation instead of forcing the replacement.

### Acceptance
1. The visible chart tabs remain `分时图 / 日K图 / 月K图 / 年K图`.
2. Existing parent-facing data/event contract remains compatible.
3. The new adapter exposes clear overlay/indicator registration seams.
4. Frontend unit tests pass.
5. `npm --prefix frontend run build` passes.
6. Browser validation confirms chart switching works without breaking Dev1's trading-plan workflow.

## ZH
### 摘要
已为 Dev2 制定 GOAL-012-R1 之后的下一步任务。当前 PM 指令已经明确，不再继续泛化调研，而是实际推动图表引擎替换。结合已完成调研和当前仓库结构，唯一值得继续推进的替换候选是 `klinecharts`；同时迁移必须严格限制在现有 `frontend/src/modules/stocks/charting/**` 适配层边界内，确保不干扰 Dev1 正在推进的 Step 4.2 之后主线工作。

### 为什么是这一步
- `TradingView-vue` 不适合当前 Vue 3 + Vite 工程。
- `vue-tradingview-widgets` 更适合嵌入 widget，不适合作为第一方 `minuteLines` / `kLines` 主图内核。
- `vue3-apexcharts` 更偏通用报表，不是股票终端主图优先方案。
- `klinecharts` 是唯一具备继续替换试验价值的候选，且强调内置指标、绘图模型和后续扩展能力。
- 当前仓库已经有安全替换边界：
  - `frontend/src/modules/stocks/StockCharts.vue`
  - `frontend/src/modules/stocks/charting/chartViews.js`
  - `frontend/src/modules/stocks/charting/useStockChartAdapter.js`

### 范围边界
Dev2 允许修改：
- `frontend/src/modules/stocks/StockCharts.vue`
- `frontend/src/modules/stocks/charting/**`
- `frontend/package.json`
- 必要的前端测试文件

Dev2 禁止修改：
- 后端 `TradingPlan` 实体与 API 流
- Step 4.2 的 draft/save 逻辑
- `StockAgentAnalysisHistory` contract
- Step 4.3-4.5 的后端方案

### 实施计划
1. 以精确版本方式引入 `klinecharts`。
2. 保持父组件对外 contract 不变：
   - `minuteLines`
   - `kLines`
   - `basePrice`
   - `interval`
   - `aiLevels`
   - `update:interval`
3. 在 `charting/**` 下新增专用 `klinecharts` 适配器，不允许把图表引擎逻辑重新塞回 `StockCharts.vue`。
4. 保持现有单主图 Tab 体验：
   - `分时图`
   - `日K图`
   - `月K图`
   - `年K图`
5. 第一阶段先迁移现有能力：
   - 分时主图
   - K 线主图
   - AI 支撑/突破叠加线
   - 当前 hover / crosshair 信息
6. 第二阶段补清晰扩展点：
   - overlay registry
   - indicator registry
   - marker / signal layer API
7. 最终输出一份结论回执，说明 `klinecharts` 是否可正式替换生产引擎，还是应保持为受控试验并回退。

### 风险
- 当前 npm 元数据显示 `klinecharts` 最新版本为 `10.0.0-beta1`，因此必须锁定精确版本，并保留回退纪律。
- 若分钟图行为、交互一致性或 overlay 扩展性不如当前实现，Dev2 必须在回执中给出回退建议，不能为了“换库”而强推上线。

### 验收
1. 图表按钮仍为 `分时图 / 日K图 / 月K图 / 年K图`。
2. 现有父层数据与事件 contract 继续兼容。
3. 新适配层暴露清晰的 overlay / indicator 注册入口。
4. 前端单测通过。
5. `npm --prefix frontend run build` 通过。
6. 浏览器交互验证确认切图正常，且不影响 Dev1 交易计划主线。
