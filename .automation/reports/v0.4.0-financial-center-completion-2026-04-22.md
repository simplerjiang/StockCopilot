### P0 Blockers Closed in V040-S6

- **B-1** Detail drawer's 3-statement overview rendered all `—`. Root cause: backend `ths` channel returns Chinese keys, some prefixed with `*` (e.g. `*资产合计`, `*净利润`); the dictionary only had English keys. Fix: extended `financialFieldDictionary.js` fallbacks with the actual Chinese keys, and extended `pickFieldValue` to transparently strip the leading `*` when indexing dict keys. 14/15 fields now populate; gross-profit still empty pending V040-S6-FU-1.
- **B-2** Keyword input did not narrow the list. Root cause: backend `GET /api/stocks/financial/reports` does not implement keyword matching today. Fix: in `useFinancialCenterQuery.js` introduced a frontend secondary filter on `symbol`/`name`. When keyword is non-empty, the request is forced to `page=1, pageSize=max(current, 100)` so the secondary filter has a wider window. Real backend full-text matching is V040-S6-FU-2.
- **B-3** Window title and About box still showed `0.3.4`. Fix: bumped `Directory.Build.props`, `frontend/package.json` (+ lock), `scripts/windows-installer.iss`, `scripts/build-windows-installer.ps1` to `0.4.0`.

### Known Issues / Backlog

- **V040-S6-FU-1** (high, v0.4.1): Detail drawer's `毛利润` (gross profit) shows `—`. Backend `ths` channel does not return `grossProfit`, nor `*营业总成本` to derive it. Two paths: (a) backend computes `revenue - 营业总成本` at collection time and stores it, or (b) frontend `pickFieldValue` learns to evaluate a tiny expression fallback. Recommend (a) for consistency with PDF-derived data.
- **V040-S6-FU-2** (high, v0.4.1): Frontend keyword filter only sees the current `pageSize` window (capped at 100 in keyword mode). When the underlying match-set exceeds 100 rows, results are silently truncated. Backend should match `keyword` against `symbol`/`name`/`reportTitle` server-side.
- **V040-S3-FU-1** (medium, v0.4.1): Channel tag colors are inconsistent across the list table and the collection panel/drawer (e.g. `ths` is green in the table, purple in the panel). `未分类/Unknown` pill design also TBD. Single source-of-truth color map needed.
- **V040-DEBT-4** (candidate, v0.4.1): Backend `StockSearchService` has no market filter and no deterministic ordering. Tencent `s3` returns both `sh000001` (index) and `sz000001` (PA Bank) for `000001`; consumers that take `data[0]` blindly get the wrong record. Frontend Financial Center already mitigates via `pickStockMatch`, but other consumers (stock-detail page) remain exposed.

### Lessons Learned

- **PM boundary violation**: Earlier in this sprint a Dev Agent ran `git commit --amend` + force-push, causing SHA drift the PM had already published. From now on Dev Agent never amends and never pushes — that decision belongs to PM.
- **Dictionary design must follow real backend keys, not English standards**: `financialFieldDictionary.js` v1 used English `key` + English `fallbacks` based on the documented schema. The actual `ths` channel returns Chinese keys (and a `*`-prefixed subset). The fix is cheap, but the principle is: when introducing a translation layer, sample 1 real payload from every channel before locking the schema.
- **Keyword should have been backend from day 1**: shipping the input box without backend support produced an obvious P0. Lesson: every visible filter must have a documented backend contract before the UI ships, or be explicitly disabled.

### 教训

- **PM 边界违规**：本 Sprint 早期 Dev Agent 自行 `git commit --amend` 并 force-push，导致 PM 已对外公布的 SHA 漂移。规则收紧：Dev Agent 永远不 amend、不 push，决定权交给 PM。
- **字典模式应当从一开始就考虑后端真实键名而非英文标准**：`financialFieldDictionary.js` v1 用英文 key + 英文 fallback，结果 `ths` 渠道全是中文键还带 `*`。修起来不贵，但教训是：引入翻译层前先把每个渠道的真实 payload 抽 1 条样本。
- **关键词应当从一开始就走后端**：先把输入框上线再补后端，导致一个明显的 P0。规则：每个可见的筛选必须有明确的后端契约，或者就显式禁用。
