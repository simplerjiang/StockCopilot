# GOAL-013 Step 2.4 Development Report (2026-03-13)

## EN
### Scope
- Replaced the deprecated local keyword sentiment path with a low-cost batch AI enrichment layer for local-fact display.
- Kept stock-agent context isolated from cheap AI labels while still preserving translated headlines for downstream reasoning.

### Development
- Backend data/model changes:
  - Added `IsAiProcessed`, `TranslatedTitle`, `AiSentiment`, `AiTarget`, and `AiTags` to `LocalStockNews` and `LocalSectorReports`.
  - Added AI-processing indexes in `AppDbContext` and runtime SQL patch guards in `LocalFactSchemaInitializer`.
  - Generated EF migration `20260313074822_AddLocalFactAiFields` and applied it to the local SQL Server database.
- Backend enrichment flow:
  - Added `LocalFactAiEnrichmentService` to batch pending market/stock/sector local facts through `gemini-2.5-flash-lite`.
  - The enrichment prompt enforces JSON-only output with translation, sentiment, target, and multi-tag classification.
  - On failure or rate-limit conditions, raw news remains stored, displayed sentiment falls back to neutral, and rows remain retryable because `IsAiProcessed` stays false.
  - `LocalFactIngestionService` now preserves prior AI fields across delete/reinsert refreshes by stable-key matching, then triggers batch enrichment only for pending rows.
- Query and agent isolation:
  - `/api/news` now maps local facts from AI fields instead of the removed `LocalNewsSentimentClassifier`.
  - Added `StockAgentLocalFactProjection` so Stock Agents receive only original/translated titles, source, category, time, and URL.
  - Agent payloads no longer include `AiSentiment`, `AiTarget`, or `AiTags`.
- Frontend:
  - `StockInfoTab.vue` now renders translated local-news headlines, shows original titles when a translation exists, and exposes `AiTarget` plus `AiTags` in the market banner and side buckets.
  - Updated `StockInfoTab.spec.js` to assert translated-title and tag rendering.
- Tests added/updated:
  - Added `LocalFactAiEnrichmentServiceTests`
  - Added `StockAgentLocalFactProjectionTests`
  - Updated `QueryLocalFactDatabaseToolTests`
  - Updated `StockInfoTab.spec.js`

## ZH
### 范围
- 用低成本批量 AI 清洗层替换了已废弃的本地关键词情绪链路，供本地事实展示使用。
- 同时隔离了 Stock Agent 上下文，避免廉价 AI 标签进入高阶分析，只保留译题等可读性增强信息。

### 开发
- 后端数据/模型改动：
  - 为 `LocalStockNews` 与 `LocalSectorReports` 增加 `IsAiProcessed`、`TranslatedTitle`、`AiSentiment`、`AiTarget`、`AiTags`。
  - 在 `AppDbContext` 增加 AI 处理索引，并在 `LocalFactSchemaInitializer` 增加运行时 SQL 补丁与索引守卫。
  - 生成 EF migration `20260313074822_AddLocalFactAiFields` 并已应用到本地 SQL Server。
- 后端清洗流程：
  - 新增 `LocalFactAiEnrichmentService`，按批次把待处理的 market/stock/sector 本地事实送入 `gemini-2.5-flash-lite`。
  - Prompt 强制 JSON-only 输出，返回翻译、情绪、靶点和多标签分类。
  - 遇到失败或限流时，原始新闻仍已入库，展示默认情绪回退到 `中性`，且 `IsAiProcessed` 保持 `false` 以便下轮重试。
  - `LocalFactIngestionService` 在 delete/reinsert 刷新时通过稳定键继承旧 AI 字段，只对待处理行触发批量清洗。
- 查询与 Agent 隔离：
  - `/api/news` 已改为读取 AI 字段，不再依赖已删除的 `LocalNewsSentimentClassifier`。
  - 新增 `StockAgentLocalFactProjection`，使 Stock Agent 只拿到原题/译题、来源、分类、时间和 URL。
  - Agent 载荷不再包含 `AiSentiment`、`AiTarget`、`AiTags`。
- 前端：
  - `StockInfoTab.vue` 现在展示本地新闻译题；存在译题时显示“原题”；同时展示 `AiTarget` 与 `AiTags`，覆盖大盘条带和侧边 bucket。
  - `StockInfoTab.spec.js` 已补充译题和标签渲染断言。
