# GOAL-012-R3 Development Report / 开发报告

## EN
- Task: Start GOAL-012-R3 by replacing the hardcoded legend model with a unified chart strategy registry behind `frontend/src/modules/stocks/charting/**`, while keeping the parent `StockCharts.vue` contract unchanged.
- Code changes:
  - Added `frontend/src/modules/stocks/charting/chartStrategyRegistry.js` as the Phase A strategy registry foundation. It now owns per-view strategy definitions, grouped chip metadata, visibility defaults, render-plan aggregation, and the first custom indicator registrations.
  - Added `frontend/src/modules/stocks/charting/chartPanes.js` to centralize pane IDs for the registry and renderer.
  - Simplified `frontend/src/modules/stocks/charting/chartViews.js` so views no longer hardcode legend chips.
  - Refactored `frontend/src/modules/stocks/charting/klinechartsRegistry.js` to consume normalized render plans instead of hand-written `MA/VOL/AI/base-line` branches.
  - Updated `frontend/src/modules/stocks/charting/useStockChartAdapter.js` to register custom strategies, build render plans per active view, and pass those plans into the chart renderer.
  - Updated `frontend/src/modules/stocks/StockCharts.vue` to render grouped strategy controls by category (`基础图层 / 趋势策略 / 动量指标 / 信号标记`) using the shared per-view visibility state.
  - Extended `frontend/src/modules/stocks/StockCharts.spec.js` with grouped-control and registry-driven coverage for `MA60`, `BOLL`, `VWAP`, and `ORB`, while keeping existing chart behavior tests green.
- Implemented strategy set in this phase:
  - Core: `price`, `volume`, `AI 价位`, `昨收基线`
  - Trend: `MA5`, `MA10`, `MA20`, `MA60`, `VWAP`, `BOLL`, `Donchian`
  - Oscillator: `MACD`, `RSI`, `KDJ`
  - Signal: `ORB`
- Validation:
  - Command: `npm run test:unit -- src/modules/stocks/StockCharts.spec.js src/modules/stocks/StockInfoTab.spec.js`
  - Result: pass (45/45)
  - Command: `npm run build`
  - Result: pass
  - Command: Browser MCP on backend-served `http://localhost:5119/`
  - Result: loaded `sz000021`, verified grouped K-line controls, toggled `MA60` and `MACD`, switched to `分时图`, and verified `VWAP` plus `ORB` active-state toggles; fresh browser console errors remained `0`
- Issues / risks:
  - `ATR`, `TD Sequential`, cross-family markers, breakout/fake-breakout, gap, divergence, and VWAP strength markers are still pending later GOAL-012-R3 phases.
  - The startup script rerun was sensitive to terminal directory drift; absolute-path invocation avoided the false “file not found” failure.
- Conclusion:
  - GOAL-012-R3 has started successfully. Phase A is now landed with a working grouped registry model and an initial real strategy batch validated in unit tests, production build, and backend-served Browser MCP.
