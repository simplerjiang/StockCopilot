# GOAL-013 Step 2.4 Planning Report (2026-03-13)

## EN
### Summary
Planned Step 2.4 to replace the inaccurate local keyword sentiment layer with a low-cost batch LLM cleaning pass before local-fact display, while keeping high-order stock-agent prompts free from cheap AI labels.

### Planned Scope
- Add AI enrichment fields to `LocalStockNews` and `LocalSectorReports`: `IsAiProcessed`, `TranslatedTitle`, `AiSentiment`, `AiTarget`, `AiTags`.
- Add a batch enrichment service using provider `openai` and model `gemini-2.5-flash-lite`, processing pending local facts in 5-20 item batches.
- Keep ingestion resilient: raw news must still be stored when LLM calls fail, default displayed sentiment stays neutral, and failed rows remain pending for retry.
- Switch `/api/news` display DTOs to use AI-cleaned translation/sentiment/target/tags.
- Prevent anchoring pollution by projecting local facts for Stock Agents without `AiSentiment`, `AiTarget`, or `AiTags`.
- Update frontend local-news rendering so translated titles and AI tags/targets are visible in the market banner and side buckets.
- Generate EF migration, apply it to the local database, and verify new columns with SQLCMD.

### Planned Validation
1. `dotnet test backend/SimplerJiangAiAgent.Api.Tests/SimplerJiangAiAgent.Api.Tests.csproj --filter "LocalFactAiEnrichmentServiceTests|QueryLocalFactDatabaseToolTests|StockAgentLocalFactProjectionTests"`
2. `dotnet test backend/SimplerJiangAiAgent.Api.Tests/SimplerJiangAiAgent.Api.Tests.csproj`
3. `npm run test:unit -- src/modules/stocks/StockInfoTab.spec.js`
4. `npm run test:unit`
5. `npm run build`
6. `dotnet ef migrations add AddLocalFactAiFields`
7. `dotnet ef database update`
8. `sqlcmd` column verification for `LocalStockNews` and `LocalSectorReports`

### Risks Noted
- Existing delete/reinsert local-fact upserts could accidentally drop AI enrichment state unless matched-field carry-over is implemented.
- The UI and multi-agent pipeline share local-fact payloads today, so agent-safe projection is required to avoid leaking cheap labels.
- The non-interactive terminal keeps cwd between commands, so root/frontend command boundaries must be explicit.

## ZH
### 摘要
规划执行 Step 2.4：在本地事实展示链路中，用低成本批量 LLM 清洗替代不准确的关键词情绪判断；同时保证高阶 Stock Agent 提示词不被廉价 AI 标签污染。

### 规划范围
- 为 `LocalStockNews` 与 `LocalSectorReports` 增加 AI 清洗字段：`IsAiProcessed`、`TranslatedTitle`、`AiSentiment`、`AiTarget`、`AiTags`。
- 新增批量 AI 清洗服务，通过 provider `openai` + model `gemini-2.5-flash-lite`，按 5-20 条批量处理待清洗本地事实。
- 保证采集主流程稳健：即使 LLM 失败，原始新闻仍正常入库，展示默认情绪保持 `中性`，失败行保持待处理以便下轮重试。
- 将 `/api/news` 返回 DTO 切换为使用 AI 清洗后的翻译、情绪、靶点与标签。
- 为防止锚定污染，Stock Agent 使用的本地事实需经单独投影，剥离 `AiSentiment`、`AiTarget`、`AiTags`。
- 更新前端本地新闻渲染，在大盘条带和侧边 bucket 中展示译题与 AI 标签/靶点。
- 生成 EF migration，应用到本地数据库，并通过 SQLCMD 校验新字段。

5. `npm run build`

8. 使用 `sqlcmd` 校验 `LocalStockNews` 与 `LocalSectorReports` 列结构

### 风险记录
- 现有 delete/reinsert 式本地事实 upsert 如果不做匹配继承，容易丢失已清洗的 AI 状态。
- 目前 UI 和多 Agent 复用了本地事实载荷，必须增加 agent-safe 投影层以隔离廉价标签。
- 非交互终端会复用 cwd，前后端命令切换时必须显式确认目录。
