# GOAL-013 Step 2.2 Development Report (EN + ZH)

## EN
### Summary
Completed GOAL-013 Step 2.2. Fixed the terminal layout stretch caused by the AI sidebar, made chart resizing resilient when the sidebar is collapsed or expanded, and hardened OpenAI/Gemini transport diagnostics with base-url normalization and clearer network error reporting.

### Changes Delivered
- Updated `TerminalView.vue` so the left terminal no longer inherits the right sidebar height: removed `min-height: 100%`, added sticky positioning, `height: max-content`, and explicit shrink-safe `min-width: 0; min-height: 0` on nested containers.
- Refactored `StockCharts.vue` resize handling to use wrapper refs plus `requestAnimationFrame` scheduling, and observed both wrappers and chart nodes so Lightweight Charts recomputes after sidebar width changes.
- Added additional shrink-safe CSS to the grid/flex tree around chart canvases to prevent overlap and overflow when the workspace switches between normal and focused layout.
- Hardened `OpenAiProvider.cs` by normalizing OpenAI-compatible base URLs (auto-append `/v1`, trim duplicated `/chat/completions`), logging full request URI plus sanitized headers, and surfacing readable `HttpRequestException` / timeout diagnostics.
- Extended frontend and backend regression coverage for chart resize reactions and OpenAI provider URL/network-failure behavior.
- Extended the existing Edge validation script to click the AI sidebar toggle, verify terminal/chart width changes in both directions, and fail on chart overflow beyond wrapper bounds.

## ZH
### 摘要

### 本次交付
- 调整 `TerminalView.vue`，移除导致高度绑定的 `min-height: 100%`，改为 sticky 终端布局，并为嵌套容器补齐 `min-width: 0; min-height: 0`，避免左侧主视区被右栏高度拖长。

- 在图表周边 Grid/Flex 树补齐收缩约束，防止 `<canvas>` 因默认最小尺寸导致越界或相互遮挡。
- 强化 `OpenAiProvider.cs`：自动补齐 OpenAI 兼容 BaseUrl 缺失的 `/v1`，去重错误拼接的 `/chat/completions`，记录完整请求 URI 与脱敏请求头，并将 `HttpRequestException` / 超时包装为可读性更强的异常信息。
- 补充前后端回归测试，覆盖图表响应式重算以及 OpenAI provider 的 URL 归一化与网络失败报错。
- 扩展现有 Edge 校验脚本，实际点击 AI 侧栏开关，验证主终端和图表宽度能随之放大/缩小，并在图表越界时直接失败。
