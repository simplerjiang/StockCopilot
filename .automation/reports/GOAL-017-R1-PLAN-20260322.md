# GOAL-017-R1 Plan / 量化归一化输入层与 Contract 详细设计

## Plan (EN)
- Objective: define the first executable slice of GOAL-017 by locking the normalized market-data layer and the shared quant contracts before any real Skender or Lean integration starts.
- Decision: R1 remains contract-first.
  - no real Skender package integration yet
  - no real Lean runtime integration yet
  - no new front-end feature yet
  - focus on DTOs, service boundaries, compatibility mapping, and acceptance rules
- Main outputs:
  - `NormalizedBar` / `NormalizedBarSeries`
  - `QuantEngineDescriptorDto`
  - `QuantFeatureSnapshotDto`
  - `QuantStrategySignalDto`
  - `QuantEngineComparisonDto`
  - `AgentQuantContextDto`
- Compatibility strategy: evolve from existing `StockCopilot*Dto` models instead of discarding them, so MCP and chart-related work can migrate incrementally.

## 计划（ZH）
- 目标：把 GOAL-017 的第一执行切片固定下来，在真正接入 Skender 或 Lean 之前，先锁定归一化行情输入层与共享量化 contract。
- 决策：R1 仍然是 contract-first。
  - 暂不真正接入 Skender 包
  - 暂不真正接入 Lean runtime
  - 暂不新增前端 UI 功能
  - 重点是 DTO、服务边界、兼容映射与验收规则
- 主要输出：
  - `NormalizedBar` / `NormalizedBarSeries`
  - `QuantEngineDescriptorDto`
  - `QuantFeatureSnapshotDto`
  - `QuantStrategySignalDto`
  - `QuantEngineComparisonDto`
  - `AgentQuantContextDto`
- 兼容策略：不推倒现有 `StockCopilot*Dto`，而是做渐进式映射，让 MCP 与图表相关工作可以平滑迁移。

## Risks / Notes (EN)
- The main risk in R1 is over-designing DTOs before verifying the migration path from existing MCP models; the plan therefore keeps explicit backward-compatibility mapping as a required deliverable.
- Another risk is letting comparison contracts leak into the default user-visible path. R1 locks the rule that comparison results are developer/replay/calibration artifacts, not default chart payload.

## 风险 / 备注（ZH）
- R1 的主要风险是 DTO 设计过度理想化，却没有验证与现有 MCP model 的迁移路径；因此本规划把向后兼容映射列为必交付项。
- 另一个风险是让 comparison contract 提前渗入默认用户视图；R1 明确锁定：comparison 结果只服务开发者、回放与校准，不进入默认图表载荷。
