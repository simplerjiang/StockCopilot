# GOAL-013 Step 2.6 Planning Report (2026-03-13)

## EN
### Summary
Planned Step 2.6 to clean polluted market-news inputs, replace dead overseas RSS feeds with verified active ones, and add a searchable archive workspace for all AI-processed local facts.

### Planned Scope
- Switch the Sina market roll source from the mixed social/news stream to the finance-only `pageid=153&lid=2509` feed.
- Add source-level filtering so any market item whose `Source` contains low-quality markers such as `自媒体` is dropped before persistence.
- Replace stale WSJ/NYT RSS entries with five verified active feeds: CNBC, Seeking Alpha, CoinTelegraph, TechCrunch, and The Hill.
- Keep the existing recency policy intact instead of relaxing age limits for stale external feeds.
- Add `GET /api/news/archive` with keyword, level, sentiment, and pagination support over both `LocalStockNews` and `LocalSectorReports`.
- Add a dedicated `全量资讯库` Vue tab with structured archive rendering, translated-title priority, raw-title secondary text, sentiment/target/tag badges, and source jump-outs.
- Extend unit tests and browser validation so the new backend query path and UI archive interactions are covered.

### Source Validation
Validated from the terminal before implementation:
- `https://feed.mix.sina.com.cn/api/roll/get?pageid=153&lid=2509&num=3&versionNumber=1.2.8.1` -> `200`, JSON payload
- `https://www.cnbc.com/id/10000664/device/rss/rss.html` -> `200`, XML
- `https://seekingalpha.com/feed.xml` -> `200`, XML
- `https://cointelegraph.com/rss` -> `200`, XML
- `https://techcrunch.com/feed/` -> `200`, RSS/XML
- `https://thehill.com/news/feed` -> `200`, RSS/XML

### Planned Validation
1. `dotnet test backend/SimplerJiangAiAgent.Api.Tests/SimplerJiangAiAgent.Api.Tests.csproj --filter "FullyQualifiedName~LocalFactIngestionServiceTests|FullyQualifiedName~RssMarketNewsParserTests|FullyQualifiedName~QueryLocalFactDatabaseToolTests"`
2. `npm --prefix frontend run test:unit -- src/modules/stocks/NewsArchiveTab.spec.js src/modules/stocks/StockInfoTab.spec.js`
3. `npm --prefix frontend run build`
4. `node frontend/scripts/edge-check-goal013.mjs`

### Risks Noted
- The archive API must merge two local-fact tables without drifting field semantics between stock and sector/market rows.
- UI filtering needs to remain usable even when the archive contains many dense AI-tagged rows.
- Replacing RSS sources must not accidentally hide all market news when one or more upstream feeds fail intermittently.
