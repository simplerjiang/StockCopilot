# v0.5.x — Baostock.NET 数据源接入 / 多源数据融合 / AI 分析增强

> **使命**：将自研的 Baostock.NET（纯 .NET 9，27 个 API，免费数据源）接入系统，填补现有数据基础设施的关键空白——指数成分、交易日历、宏观经济、行业分类——并为 AI 分析、市场情绪分析、财报中心提供更丰富、更可靠的数据底座。

---

## 1. 背景与动机

### 1.1 现有数据基础设施现状

| 数据类别 | 当前来源 | 实时性 | 已知问题 |
|---------|---------|--------|---------|
| 股票行情/K线 | 东方财富 push2（主）+ 腾讯/新浪/百度（备） | 实时 | 单一体系，东财 CDN 偶尔封锁 |
| 财务报表 | 东财 emweb → 东财 datacenter → 同花顺 | 季频 | 三源均为东财系，多样性不足 |
| 市场情绪 | 东财板块资金流 + 论坛帖量 + RSS 新闻 | 日频/实时 | 缺乏宏观经济定量指标 |
| 交易日历 | 硬编码节假日列表 `KnownHolidayClosures` | 静态 | 每年需手动维护，易遗漏 |
| 指数成分 | **无** | — | 无法做指数相对分析 |
| 宏观经济 | 仅通过新闻关键词（央行、利率、GDP） | 新闻级 | 无结构化时间序列数据 |
| 行业分类 | 东财报价字段 `SectorName` | 静态 | 非证监会标准分类 |
| ST/退市追踪 | **无** | — | 推荐系统缺少风险过滤 |

### 1.2 Baostock.NET 能提供什么

Baostock.NET 是我们自研的纯 .NET 9 客户端（MIT 许可），对接 baostock.com 免费 A 股数据服务，27 个 API，全异步 `IAsyncEnumerable<T>` 流式返回，强类型 record 模型。

| API 分类 | 接口数 | 数据范围 | 对项目的价值 |
|---------|-------|---------|-------------|
| K 线/行情 | 2 | 日/周/月/分钟 K 线 | 🟡 备用数据源 + 交叉验证 |
| 季频财务（6 类） | 6 | 盈利/营运/成长/杜邦/资产负债/现金流 | 🔴 **高价值** — 独立数据源交叉验证 |
| 估值/分红/复权 | 2 | 股息分红 + 复权因子 | 🟡 补充现有分红数据 |
| 公司公告 | 2 | 业绩快报 + 业绩预告 | 🔴 **高价值** — 财报中心新维度 |
| 宏观经济 | 5 | 存贷款利率/准备金率/货币供应量 | 🔴 **高价值** — 填补空白 |
| 基础信息/日历 | 3 | 交易日历/证券列表/证券基本资料 | 🔴 **高价值** — 替代硬编码 |
| 板块/指数成分 | 4 | 行业分类/沪深300/上证50/中证500 | 🔴 **高价值** — 填补空白 |
| .NET 扩展 | 4 | 终止上市/暂停上市/ST/*ST | 🟠 **中高价值** — 风险过滤 |

## 2. 价值分析：对三大核心模块的影响

### 2.1 AI 分析增强

| 增强点 | Baostock 数据 | 当前短板 | 预期效果 |
|-------|-------------|---------|---------|
| **宏观-微观联动分析** | 存贷款利率 + 准备金率 + M1/M2 货币供应量 | AI 只能从新闻文本推断宏观环境 | AI 可基于结构化宏观时间序列做定量分析 |
| **指数相对强弱** | HS300/SZ50/ZZ500 成分股列表 | 无法判断个股相对指数表现 | AI 可计算 α/β、跑赢/跑输指数 |
| **行业轮动精准化** | 证监会标准行业分类 | 东财板块分类不稳定 | 行业轮动分析基于标准化分类，结果更可靠 |
| **财务数据交叉验证** | 独立的季频六维财务数据 | 财务数据仅依赖东财体系 | 两源对比可发现异常数据/API 故障 |
| **风险预警** | ST/*ST/退市/暂停上市列表 | 推荐系统无法自动排除风险股 | 自动标记高风险证券 |

### 2.2 市场情绪分析增强

| 增强点 | Baostock 数据 | 当前短板 | 预期效果 |
|-------|-------------|---------|---------|
| **货币政策量化** | 存贷款利率变动 + 准备金率调整历史 | 仅通过新闻关键词感知央行动向 | 利率/准备金时间序列 → 政策松紧信号量化 |
| **流动性指标** | M1/M2 月度数据 | 完全缺失 | M1-M2 剪刀差 → 市场资金面趋势 |
| **指数成分变动** | 定期成分股调整 | 无法追踪 | 成分股纳入/剔除 = 重要情绪信号 |
| **业绩预期** | 业绩快报 + 业绩预告 | 仅有财报正式数据 | 预告/快报 → 市场预期差分析 |

### 2.3 财报中心增强

| 增强点 | Baostock 数据 | 当前短板 | 预期效果 |
|-------|-------------|---------|---------|
| **杜邦分析** | DupontRow（ROE 拆解） | 需手动从报表计算 | 一键杜邦分析：ROE = 净利率 × 周转率 × 杠杆 |
| **成长性指标** | GrowthRow（营收/利润/净资产增长率） | 需手动比较同期数据 | 直接获取 YoY 增长率时间序列 |
| **营运效率** | OperationRow（周转天数/周转率） | 需手动计算 | 现成的营运效率指标 |
| **数据完整性校验** | 两源对比 | 单源无法发现错误 | 异常检测：两源偏差 > 阈值则告警 |
| **历史分红记录** | DividendRow（完整分红历史） | 东财分红数据可靠性一般 | 分红策略分析 + 股息率计算 |

## 3. v0.5.x 版本规划

### 3.0 总体原则

1. **Baostock 是补充源，不是替代源** — 东财实时行情仍为主源，baostock 用于填补空白 + 交叉验证
2. **渐进式接入** — 先接入最高价值、零依赖的数据，再做深度集成
3. **NuGet 包可双向迭代** — Baostock.NET 在独立仓库，如需增强可同步修改
4. **数据层抽象** — 新增 `IBaostockDataService` 接口，与现有服务解耦

### 3.1 v0.5.0 — 基础管道 + 交易日历 + 指数成分

> 打通 Baostock.NET 接入管道，替换硬编码交易日历，补齐指数成分数据。

**范围：**

|-------|------|------|---------|
| S1: NuGet 集成 + 连接管理 | S | 添加 `Baostock.NET` NuGet 引用，实现 `IBaostockClientFactory`，支持连接池/重连 | 单元测试通过；集成测试能连上 baostock 服务器 |

> 全量接入宏观五维数据，重点解决两个问题：如何快速呈现给用户，如何让 LLM 理解并使用。

**范围：**

## 4. 对 Baostock.NET NuGet 包的改进建议

在接入过程中，可能需要在 Baostock.NET 仓库做以下增强：

| 改进项 | 状态 | 优先级 |
|-------|------|-------|
| K 线解析 IndexOutOfRange | ✅ 已修复（v1.0.0 更新） | — |
| 连接池 / 多会话支持 | 已决策：放消费端实现，NuGet 包保持单连接 | — (消费端) |
| 重连 + 熔断机制 | baostock 服务偶尔不稳定 | P0 |
| 日期格式自动归一化 | 年份简写 `"2024"` 不被服务端接受，需自动转 `yyyy-MM-dd` | P1 |
| 批量查询优化 | 批量拉取多只股票财务数据时减少 TCP 往返 | P1 |
| 本地缓存层（可选） | 高频调用相同数据时避免重复网络请求 | P1 |
| `ITransport` Mock 工厂 | 方便消费端写集成测试 | P2 |
| 查询超时配置 | 不同查询类型可设置不同超时 | P2 |

## 6. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| baostock.com 服务不稳定/停服 | 数据断供 | baostock 仅作补充源；关键路径仍走东财；本地缓存兜底 |
| 数据延迟（baostock 非实时） | 宏观/财务数据有滞后 | 明确标注数据时效；不用于实时决策 |
| TCP 长连接在 NAT/防火墙后断开 | 查询失败 | NuGet 包增加重连机制 + 消费端熔断 |
| v0.4.x 尚未完成，v0.5.x 过早启动 | 资源冲突 | v0.5.0 在 v0.4.x 基本稳定后启动 |
| 两源数据口径不一致 | 交叉验证误报 | 建立字段映射表；允许配置容差阈值 |
| 部分宏观 API 数据量有限 | 存贷款利率/准备金率历史数据仅数条 | 扩大查询范围（2010+）；如数据不足，M1/M2 货币供应量作为主要宏观指标 |

## 11. 技术决策落档

> **本节为 v0.5.x 技术方案的权威记录。** 经 PM 与用户讨论确认，2026-04-23。

### 11.1 连接管理：连接池放消费端

- Baostock.NET NuGet 包只提供单连接 `BaostockClient`（现状保持不变）
- 本项目（消费端）实现 `BaostockClientPool`，基于 `Channel<BaostockClient>` 的简单连接池
- DI 注册为 `Singleton`，最大连接数 3，空闲超时 5 分钟
- **理由**：连接池逻辑跟业务场景强相关，放消费端更灵活

```csharp
// Program.cs
services.AddSingleton<IBaostockClientFactory, BaostockClientFactory>();
// 内部维护连接池，不修改 NuGet 包
```

### 11.2 LLM Context 注入：通过现有 MCP Tool 体系

- **不修改现有 prompt/tool 内容**，仅新增 `macro_indicators` tool
- 宏观数据通过 `RecommendToolDispatcher` 注册为新 tool，MacroAnalyst 角色可调用
- 返回结构化 JSON（含 current/direction/percentile/signal 等字段）
- 预生成 `policy_summary` 一句话总结，减少 LLM 推理负担

```json
{
  "tool": "macro_indicators",
  "result": {
    "snapshot_date": "2026-04-23",
    "lpr_1y": { "current": 3.10, "direction": "下行", "percentile_10y": 12, "signal": "偏宽松" },
    "m2_yoy": { "current_pct": 8.7, "trend_3m": "持续上行", "percentile_10y": 45 },
    "m1_m2_spread": { "current": -4.2, "direction": "收窄", "interpretation": "资金活化" },
    "rrr": { "current": 9.0, "direction": "下行", "signal": "释放流动性" },
    "policy_summary": "当前货币政策整体偏宽松..."
  }
}
```

### 11.3 交叉验证引擎：统一 5% 阈值

- 全字段统一 5% 偏差阈值，不分级
- 偏差基准用东财值（主源）作分母
- ROE 对比需注意口径差异（东财加权 ROE vs baostock 平均 ROE），此字段可配置为排除或单独阈值
- 校验结果存 `FinancialValidationAlert` 表，管理后台可查
- **不干预前端展示**，仅后台告警

### 11.4 ST/退市处理：仅前端标记，不排除

- **推荐系统不排除** ST/退市股票，保持现有推荐逻辑不变
- 前端股票名称旁显示风险标签：🔴 ST / ⚠️ *ST / ⛔ 退市 / ⏸ 暂停上市
- 每日采集 ST/*ST/暂停/终止上市列表，存入 `StockRiskTag` 表
- 风险标签数据通过现有股票信息 API 附加返回

### 11.5 数据采集调度

| 数据类型 | 采集频率 | 时间窗口 | 失败策略 |
|---------|---------|---------|----------|
| 交易日历 | 年初一次 + 每月校验 | 任意 | 重试 3 次；用上次缓存 |
| 指数成分 | 每月第一个交易日 | 18:00-23:00 | 重试 3 次 |
| 行业分类 | 每季度一次 | 任意 | 重试 3 次 |
| 宏观经济 | 每日 18:00 | 18:00-20:00 | 重试 3 次；数据不变跳过 |
| 季频财务 | 随东财 FinancialWorker | 15:30-23:00 | 跟随现有逻辑 |
| ST/退市列表 | 每日 | 9:00 前 | 重试 3 次 |

- 新增 `BaostockDataWorker : BackgroundService`，统一管理采集
- 熔断：连续 5 次失败 → 暂停 1 小时 → 自动恢复
- 不影响东财主源运行
