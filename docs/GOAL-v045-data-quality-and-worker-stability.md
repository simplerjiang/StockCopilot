# GOAL v0.4.5 — 数据质量与 Worker 稳定性

> 来源：`.automation/buglist.md` 20260423 人工测试 Bug 清单

## 背景

v0.4.4 完成产品质量修复后，人工测试发现 5 个新问题，集中在数据准确性和系统稳定性两个方向。

## Bug 列表

### Bug #1: 财报数值单位不一致（P0）

**现象**：财报中心 → 详情，PDF 来源显示营业收入 1700 亿，同花顺来源显示 1000 多万。两者均看起来不正确。

**分析方向**：
- PDF 解析的数值单位是否统一？（元 vs 万元 vs 亿元）
- 同花顺 API 返回的数值单位与 PDF 是否一致？
- `FinancialTableParser` 是否正确处理了"万元"/"元"表头单位？
- `FinancialDataReadService` 在汇总时是否做了单位转换？

**验收标准**：同一公司同一年度，不同来源渠道的营业收入数值一致（允许 <1% 差异）。

### Bug #2: cninfo PDF 采集失败（P1）

**现象**：来源为同花顺的数据，其 PDF 获取来自 cninfo，但采集失败。

**分析方向**：
- cninfo 网站是否可达？是否有反爬机制？
- PDF 下载 URL 是否过期或格式变更？
- 是否需要更新 headers/cookies？

**验收标准**：cninfo PDF 下载成功率 ≥ 80%。

### Bug #3: 采集中心日期选择器丢失（P1）

**现象**：采集中心的起始时间日期下拉功能丢失。

**分析方向**：
- 前端组件是否被误删或隐藏？
- 是否因条件渲染逻辑变化导致不显示？

**验收标准**：采集面板显示日期选择器，可选择起始时间。

### Bug #4: 财报中心集成采集面板（P2/功能需求）

**现象**：用户期望在财报中心直接触发采集，不用切换到采集面板。

**设计方向**：
- 财报中心页面增加"采集"入口按钮
- 点击后展开内联采集面板或弹窗
- 复用现有采集逻辑，UI 重新设计

**验收标准**：财报中心页面可直接触发指定股票的财报采集。

### Bug #5: Worker 自动关停，需自动重启（P0）

**现象**：采集 Worker 经常自动关停，需要自动检测并重启。

**分析方向**：
- Worker 为何关停？是崩溃还是正常退出？
- API 的 Worker Supervisor 是否有自动重启逻辑？
- 当前健康检查机制是否足够？

**验收标准**：Worker 异常退出后 30 秒内自动重启；连续失败 3 次后告警。
